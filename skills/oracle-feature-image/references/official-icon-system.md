# Official Icon System

> Parent skill: [[Reference/Skills/oracle-feature-image/SKILL|oracle-feature-image]]


## Source of Truth

When a story depends on Oracle AI Database or Oracle developer branding, use official assets first.

Asset source hierarchy:
- official Oracle icon library or Figma file supplied by the brand owner
- official PowerPoint, SVG, PNG, or slide deck asset pack supplied by the team
- faithful vector simplification traced from an approved Oracle asset
- abstract geometric fallback only when no approved product icon exists

Do not invent replacement logos or synthetic Oracle product symbols when a real Oracle mark or icon exists.

## Deck-Derived Style

The approved Oracle AI Database developer icon deck uses a consistent editorial system:
- one icon per card
- dark green rounded-square cards with generous whitespace
- white or soft-gray linework
- restrained coral or Oracle Red highlights
- minimal secondary gray fills
- quiet Oracle red logo tile in the corner
- optical centering and clean symmetry
- no noisy textures, fake dashboards, or oversized glows

## Icon Families

- chip + orbit or chip + star: AI core, agent intelligence, memory core, acceleration
- document + loop arrow: documents, SQL, retrieval, iteration, content flow
- gauge variants: quick start, free tier, speed, efficiency, cost
- shield or check variants: validation, trust, governance, reliability
- database + sparkle: Oracle AI Database, durable memory, managed intelligence
- database + geometry: structure, vector search, modeling, platform depth
- open box + chart: packaging, deploy-anywhere stories, delivery, platform distribution
- database + shield: protected storage, governance, secure memory

## Choosing an Official Motif

- If the article centers on Oracle AI Database, start with a database variant.
- If it centers on memory or AI core concepts, start with chip-orbit or database-sparkle.
- If it centers on trust, validation, or guardrails, start with shield or check variants.
- If it centers on docs, SQL, retrieval, or workflow, start with document-loop.
- If it centers on quick start, free tier, or efficiency, start with gauge variants.
- If it centers on deployment or packaging, start with the open-box family.

## Simplification Rules

- Prefer reusing one full official icon card or one faithful extraction of the icon itself.
- If adapting an official icon, keep its silhouette, stroke language, accent placement, and color balance intact.
- Remove slide furniture before redrawing the icon logic.
- Do not mash multiple official icons together unless the concept is unreadable without it.

## Anti-Synthetic Rules

Reject a draft if the hero visual feels like:
- a fake dashboard
- a generic neon AI chip
- an invented database glyph
- multiple floating panels or cards
- glow-heavy future-tech art
