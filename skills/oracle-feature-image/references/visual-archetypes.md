# Visual Archetypes

> Parent skill: [[Reference/Skills/oracle-feature-image/SKILL|oracle-feature-image]]


Choose one archetype and one official asset family. If two seem equally right, pick the one that stays clearer at thumbnail size and is better supported by approved Oracle iconography.

## Loop

Use for cycles, reasoning loops, orchestration, feedback, reflection, and repeated improvement.

- default move: one circular loop or orbit with a clear motion cue
- official families: chip + orbit, document + loop
- avoid: labeled multi-step rings

## Flow

Use for pipelines, sequences, handoffs, and end-to-end process explanations.

- default move: `3-4` connected forms with a directional cue
- official families: document + loop, open box + chart
- avoid: many small boxes or long labels

## Stack

Use for layered systems, model/platform/application relationships, and platform architecture.

- default move: `3-5` clean layers
- official families: database + geometry, open box, layered database variants
- avoid: full service maps

## Nodes

Use for multi-agent systems, graph relationships, networks, and connected actors.

- default move: `3-7` nodes with one dominant center or path
- official families: chip + orbit, database + geometry when the network sits inside Oracle AI Database
- avoid: dense webs that collapse at thumbnail size

## Shield

Use for safety, governance, reliability, controls, trust, and guardrails.

- default move: one shield or boundary mark with a subtle internal symbol
- official families: shield or check variants, database + shield
- avoid: compliance-dashboard visuals

## Spark

Use for launches, capability leaps, insight, acceleration, and high-level breakthrough stories.

- default move: one spark, star, or burst glyph paired with strong text
- official families: chip + star, database + sparkle
- avoid: generic sci-fi AI flourishes

## Chart

Use only when the article is truly benchmark-led and the story is about comparison or performance.

- default move: one stripped-down bar, line, or heatmap motif
- official families: gauge variants or compact official chart marks
- avoid: dashboard collage or multi-axis detail

## Database Override

If the story is specifically about Oracle AI Database, first check whether a database variant from the approved icon pack already fits. Only leave the database family if another archetype communicates the story more clearly and still feels official.

## Mapping Rules

- If the article is about a repeated cycle or reasoning loop, start with `loop`.
- If the article explains a sequence or operational path, start with `flow`.
- If the article explains layers or infrastructure, start with `stack`.
- If the article centers on multiple connected entities, start with `nodes`.
- If the article centers on security or policy, start with `shield`.
- If the article is a launch or high-level capability piece, start with `spark`.
- If benchmark results are the story, use `chart`; otherwise avoid it.
- If the article centers on Oracle AI Database persistence, vector search, semantic memory, or managed storage, start with an approved database variant before drawing a custom stack.

## Simplification Rule

If an article seems to need a full diagram:
1. Name the single concept the reader must grasp.
2. Compress the article into one symbol.
3. Remove labels before removing clarity.
4. Prefer one strong silhouette over many informative parts.

## Direction Generation

When presenting three directions:
- keep the layout stable
- vary the approved asset family or metaphor first
- keep the text system and palette stable
- name the asset family explicitly
- recommend the option with the best thumbnail clarity
